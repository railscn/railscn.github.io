Ruby on Rails 维护政策

Rails 框架支持分为四部分：新功能、问题修复、安全问题和重大安全问题。 它们的处理方式如下，除安全版本外，所有版本均采用 X.Y.Z 格式。

题要

  1. 新功能
  2. 问题修复
  3. 安全问题
  4. 重大安全问题
  5. 不支持的发行系列

Rails 遵循语义化版本使用：

补丁版本 Z

只是修复一些小问题，没有 API 修改，没有新功能。 必要的安全修复除外。

次版本 Y

增加新功能，可能包含 API 修改（可考虑作为语义版本的主版本）。 与以前的次要或主要版本中的弃用通知配对的重大更改。

主版本 X

新功能，可能会包含 API 更改。Rails 的次要版本和主要版本之间的区别在于更改的 幅度，并且通常会保留用于特定场合发布。

1 新功能

新功能仅添加到主分支，不会在点后的版本中发布。

2 问题修复

只有最新的版本系列才会收到错误修复。错误修复通常添加到主分支，如果需要，则 向后移植到最新版本系列的 x-y-stable 分支。当一个 x-y-stable 分支中添加了足够 多的错误修复时，就会从中构建一个新的补丁版本。例如，理论上的 1.2.2 补丁版本 将从 1-2-stable 分支构建。

特殊情况下，如果核心团队中的某个人同意支持更多系列，他们将被包含在支持系 列列表中。

当前包含系列： 7.1.Z.

3 安全问题

当前版本系列和下一个最新版本将收到补丁和新版本，以防出现安全问题。

这些版本是通过获取最后发布的版本、并且应用了安全补丁后而创建的。然后将这些 补丁应用到 x-y-stable 分支的最新版本。例如，理论上的 1.2.2.1 安全版本将从 1.2.2 构建，然后添加到 1-2-stable 的最新版本。这意味着如果您运行的是最新版本的 Rails， 因为是安全版本则可以轻松升级。

只有直接的安全补丁才会包含在安全版本中。由安全补丁引起的非安全相关错误的修 复可能会发布在版本的 x-y-stable 分支上，并且只会根据错误修复政策作为新 gem 发布。

当前包含系列： 7.1.Z, 7.0.Z, 6.1.Z.

4 重大安全问题

对于严重的安全问题，当前主要系列中的所有版本以及之前主要系列中的最后一个版 本都将收到补丁和新版本。安全问题的分类由核心团队判断。

当前包含系列： 7.1.Z, 7.0.Z, 6.1.Z.

5 不支持的发行系列

当发行版本不再支持时，您需要自行处理错误和安全问题。我们可能会提供修补程序 的反向移植并将它们发布到 git，但是不会发布新版本。如果您不能维护自己的版本， 则应升级到支持的版本。

反馈

衷心希望您能帮我们提升指南质量。

您发现任何错误都可以帮助我们。 如果想要参与贡献可以参阅文档贡献章节获取相关细节。

如果您发现有内容不完整或者有些内容需要更新，请进行相关补充和完善。 开始之前要检查一下 抢鲜版指南中是否 已经修复了这些问题，或者这些问题是否出现在主分支（main branch）内。 如果以上检查完成后，确认为需要完善文档，请参考《Ruby on Rails 指南》之指南， 以确保您使用了正确的编写风格和编写习惯。

如果您发现了一个问题，但是自己不会修复，请在 Github 发一个问题帖子。

最后，但并非不重要的是——Rubyonrails-Docs 邮件列表 欢迎大家提出任何关于 Ruby on Rails 文档的问题。
