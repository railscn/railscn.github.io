《Ruby on Rails 指南》之指南

本指南用于指导《Ruby on Rails 指南》的编写。本指南遵循由己及人，由人到己的原 则，本指南既是其他指南编写的指导，也是本指南本身的指导。

通过阅读本指南，您会学到：

  • Rails 文档的常用约定。
  • 如何在本地生成指南。

题要

  1. Markdown
  2. 序言
  3. 标题
  4. 链接到 API

1 Markdown

指南由 GitHub Flavored Markdown 编写而成。 这是综合性 Markdown 文档，以及备忘录。

2 序言

每一篇指南都从顶部的励志小文（就是蓝色区域的简介）开始。序言应该告诉读者指 南是关于什么的，以及他们将学到什么。例如查阅路由指南。

3 标题

每一篇指南的标题使用 h1 级标题；指南的章用 h2 标题；节用 h3 标题；注意：生成的 HTML 使用从 <h2> 开始的标题。

指南标题
===========

章
-------

### 节

编写标题时，所有单词（介词、连词、内部链接文章和动词“to be”的形式外）的首 字母大写：

#### Assertions and Testing Jobs inside Components
#### Middleware Stack is an Array
#### When are Objects Saved?

行内格式如常：

##### The `:content_type` Option

4 链接到 API

链接到 API (api.rubyonrails.org) 的链接由指南生成器按照下面方式处理：

包含发布标签的链接不做修改直接输出。例如：

https://api.rubyonrails.org/v5.0.1/classes/ActiveRecord/Attributes/ClassMethods.html

不会修改，直接输出。

请在发行说明中这样使用，因为无论生成的目标如何，它们都应指向相应的版本。

如果链接不包含发行标签，链接就会指向抢鲜版，域名指向 edgeapi.rubyonrails.org。 例如：

https://api.rubyonrails.org/classes/ActionDispatch/Response.html

变成了

https://edgeapi.rubyonrails.org/classes/ActionDispatch/Response.html

如果链接不包含发行标签，并且发行版指南已经生成，Rails 的版本就会强行介入。 例如：如果我们已经生成了 v5.1.0 的指南，那么链接

https://api.rubyonrails.org/classes/ActionDispatch/Response.html

就会变成

https://api.rubyonrails.org/v5.1.0/classes/ActionDispatch/Response.html

请不要手动链接到 edgeapi.rubyonrails.org 。

5 API 文档指导

在适当的情况下，指南和 API 应该是连贯一致的。特别是 API 文档指南 的这些部分也适用于指南：

  • 措辞
  • 英文
  • 示例代码
  • 文件名
  • 字体

6 HTML 指南

生成指南前，确保您系统内已经安装了最新版的 Bundler。在这里 可以找到最新版 Bundler。编写本文时，最新版是 v2.3 。

安装最新版 Bundler，运行 gem install bundler。

6.1 生成

要生成所有指南，需要 cd 进入 guides 目录，运行 bundle install，然后执行：

$ bundle exec rake guides:generate

或者

$ bundle exec rake guides:generate:html

生成的 HTML 文件在 ./output 目录。

只处理 my_guide.md 需要使用 ONLY 环境变量：

$ touch my_guide.md
$ bundle exec rake guides:generate ONLY=my_guide

默认情况下，没有修改的指南不会被处理，所以在实践中 ONLY 很少用到。

强迫处理所有文档，需要传递 ALL=1 参数。

如果您想要生成非英文指南，您可以将该文档放置于 source 内的一独立的文件夹（ 例如：source/es），生成时使用 GUIDES_LANGUAGE 环境变量：

$ bundle exec rake guides:generate GUIDES_LANGUAGE=es

如果您想要查看所有环境变量，您可以运行生成脚本查看配置参数：

$ rake

6.2 验证

请验证生成的 HTML：

$ bundle exec rake guides:validate

需要注意的是，标题的 ID 是基于内容生成的，所有很容易重复。

7 Kindle 格式指南

7.1 生成

生成 Kindle 格式指南，使用下面的 Rake 任务：

$ bundle exec rake guides:generate:kindle
